# nbr.haus

A modern personal portfolio website showcasing profile, projects, work history, tech stack, dev tools, AI usage, and contact information.

## Tech Stack

- **[TanStack Start](https://tanstack.com/start)** - Full-stack React framework with SSR
- **[TanStack Router](https://tanstack.com/router)** - Type-safe file-based routing
- **[React 19](https://react.dev/)** - UI framework
- **[TypeScript](https://www.typescriptlang.org/)** - Type safety
- **[CSS Modules](https://github.com/css-modules/css-modules)** - Component-scoped styling with type generation
- **[Open Props](https://open-props.style/)** - CSS design tokens
- **[Vite](https://vitejs.dev/)** - Build tool and dev server
- **[Vitest](https://vitest.dev/)** - Testing framework
- **[Lucide React](https://lucide.dev/)** - Icon library

## Getting Started

### Prerequisites

- Node.js (v18 or higher recommended)
- npm

### Installation

```bash
npm install
```

### Development

Start the development server with CSS type generation:

```bash
npm run dev
```

This runs both the dev server (port 3000) and CSS types watcher concurrently.

Alternatively, run them separately:

```bash
# Dev server only
npm run dev:server

# CSS types watcher only
npm run dev:css-types
```

Visit [http://localhost:3000](http://localhost:3000) to view the site.

### Production Build

Build the application for production:

```bash
npm run build
```

Preview the production build:

```bash
npm run serve
```

## Architecture Highlights

### Responsive Layout

The site features **four different layout variations** (1, 2, 3, and 4 columns) that adapt to viewport size using CSS. All layouts reuse the same components but arrange them differently for optimal viewing at any screen size.

### CSS Modules with Type Safety

All component styles use CSS Modules with automatically generated TypeScript definitions:

```tsx
import styles from "./Component.module.css";

export function Component() {
  return <div className={styles.container}>...</div>;
}
```

Types are generated via `typed-css-modules` and watched during development.

### Content Organization

- **Presentational components** (`src/components/`) - Reusable, dumb components
- **Content blocks** (`src/content-blocks/index/`) - Smart containers that provide content and consume presentational components

### SSR & SEO

- Server-side rendering via TanStack Start
- JSON-LD structured data for search engines
- Meta tags and Open Graph support in root layout

### Theme Customization

The site includes an `AccentPicker` component that allows theme customization via CSS custom properties. The accent hue is controlled by the `hue` search parameter (validated with Zod) and persists across navigation.

### DevTools

TanStack Router DevTools are integrated in development mode for debugging routes and navigation.

- `src/routeTree.gen.ts` is auto-generated by TanStack Router - never edit manually
- CSS Module `.d.ts` files are auto-generated - do not commit or edit manually

- Path alias `@/*` resolves to `src/*`
